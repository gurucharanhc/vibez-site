The gap

Search is literal. Travel is emotional.

Modern travel platforms still expect users to search with keywords: dates, airports, star ratings. But real trips start from a feeling: “serene river at sunrise,” “moody neon city,” “calm pastel coast”

Why keyword search fails modern travelers
Problem space
  • Literal search, emotional intent: Rigid filters cannot interpret abstract feelings or “vibes,” causing a gap between what users type and what they actually want to experience.
  • Visual disconnect: Image search is separated from text search – you can’t easily say “show me moody, cinematic places” and get both images and locations.
  • Users run out of words: People quickly fall back to generic “Top 10” lists instead of discovering unexpected places that truly match their mood.
The cost of friction

Project overview

From mood to destination in a few words.

Vibez is a research-grade capstone project exploring how multimodal models can translate subjective emotional language into concrete visual and geographic suggestions. It sits at the intersection of LLM alignment, retrieval, and UX.

What Vibez does
Problem → System

Traditional travel search expects destinations, dates, and filters. People actually start with feelings — “I want something cinematic and rainy,” “I need a bright, joyful escape.”

Vibez lets users lead with mood. A short, poetic description is mapped to:

  • Emotion labels (GoEmotions + custom vibe taxonomy)
  • Curated Unsplash imagery and color palettes
  • Location metadata (city, country, coordinates)

System design

Under the hood: data → vibes → retrieval.

Vibez is an end-to-end pipeline: we start with images and text, enrich them with emotions, generate vibe tags using LLMs, and index everything into a vector DB for fast, vibe-aware search.

1 · Curated multimodal data
Unsplash Lite + GoEmotions

We collect curated travel & lifestyle images with captions, colors, and metadata (city, country, coordinates) and align them with GoEmotions-style labels and a custom vibe taxonomy (e.g., “majestic,” “whimsical,” “intimate,” “electric”).

Features: AI image descriptions · keywords · color palettes · emotion labels.
2 · LLM-driven vibe tagging
GPT-4o + fine-tuned Mistral

GPT-4o generates synthetic vibe annotations from combined visual and textual context. These are used to fine-tune open-source models (e.g., Mistral, Gemma) to produce consistent vibe tags at scale, both in training and inference.

Goal: stable tags across prompts, templates, and LoRA checkpoints.
3 · Vibe-aware retrieval
Embeddings + vector DB

We embed vibe tags and descriptions with OpenAI embeddings and index them in Pinecone. A user query is embedded, matched against the index, and enriched with location metadata to surface safe, diverse, and on-vibe results.

Stack: semantic similarity, re-ranking, content diversity.
Technical overview

Microservices architecture on AWS.

The production-ready prototype follows a clean separation of concerns: UI, API, model layer, and vector search — all orchestrated on AWS with observability and security in mind.

System architecture
High level
  • Frontend: a web interface where users enter poetic, vibe-rich queries.
  • Backend API (FastAPI): routes requests, orchestrates retrieval, and joins results from multiple pipelines.
  • Model layer: hosted LLMs (OpenAI, fine-tuned Mistral/Gemma) serve vibe tagging, synthetic data generation, and trip planning.
  • Vector DB: Pinecone as the primary semantic search engine, also used in experimentation and offline evaluation.
Key infrastructure components
AWS layer
  • EC2 compute: GPU instances dedicated to model pre-loading and low-latency inference.
  • S3 + ECR: S3 stores curated datasets and image artifacts; ECR manages container images.
  • ALB + ACM: Application Load Balancer and TLS certificates provide secure, scalable external access.
  • CloudWatch: metrics and logs for monitoring latency, errors, and usage patterns.
External service layer
  • OpenAI: synthetic vibe generation, and trip planning.
  • Pinecone: scalable vector index for image + vibe embeddings.
  • Unsplash API: runtime image and location enrichment for retrieved records.
Application overview

Four pipelines working together.

Vibez is not a single model call. It is a collection of interconnected pipelines that take a user from a raw vibe query to images, locations, and a draft trip plan.

Input data sources
Rich & diverse
  • Unsplash Lite (25k+ photos): high-quality travel and lifestyle images, captions, keywords, and extracted color palettes.
  • GoEmotions: fine-grained emotion labels used to ground the custom vibe taxonomy in human affect.
  • Custom vibe taxonomy: a structured set of vibe words (e.g., “majestic,” “intimate,” “electric”) used consistently across tagging and retrieval.
EDA highlights
Curated data
  • Emotion patterns: identified dominant emotions and common co-occurrence pairs.
  • Color–emotion link: analyzed how warm palettes correlate with positive emotions.
  • Positive leaning: confirmed the dataset skews toward bright, optimistic feelings — ideal for travel inspiration.
Pipeline 1 · Data curation & retrieval
Images → vibes
  • Combine image features (captions, keywords, colors) with emotional labels.
  • Use GPT-4o-mini with a carefully tuned prompt to synthesize new vibe annotations.
  • Embed all enriched records into Pinecone to enable instant “emotional search.”
Pipeline 2 · Image & location search
Vector DB → Unsplash
  • Embed the user’s vibe query and retrieve candidates from the vector DB.
  • Call the Unsplash Image API to fetch high-resolution images.
  • Use the Unsplash Location API to stitch in city, country, and coordinates, turning each image into a travel-ready suggestion.
Pipeline 3 · Trip planner
LLM itinerary
  • Take keywords, vibe tags, and selected locations as structured context.
  • Use ChatGPT with a fine-tuned planner prompt to generate day-by-day ideas.
  • Return concise itineraries that stay on-vibe while remaining realistic and actionable.
Pipeline 4 · Vibe tag generator & aggregator
Fine-tuned LLM
  • Fine-tune Mistral / Gemma using supervised fine-tuning (SFT) on curated and synthetic vibe-tag pairs.
  • Generate consistent vibe tags at inference time, then aggregate them with images, locations, and planner output.
  • Feed the aggregated representation back into the UX so users see a coherent vibe story for each suggestion.

Mistral vs Gemma
Model comparison

We fine-tuned both Mistral and Gemma variants on the same curated + synthetic vibe dataset. The fine-tuned Mistral model consistently outperformed Gemma across:

  • Semantic understanding: achieving a semantic similarity score of approximately 72.3% on held-out evaluation.
  • F1 scores: over 60% higher than Gemma on multi-label vibe tagging, meaning fewer missed or spurious vibes per example.
  • Lexical similarity: closer overlap with human-written vibe tags.
Human-centric evaluation
Application & users
  • Query → tags → images: internal reviewers scored alignment between user query, generated vibe tags, and retrieved images. Results show ~97% query alignment and ~90% aesthetic quality, indicating that the system favors high-impact visuals over literal keyword matches.
  • User survey (N=22): more than 60% of respondents were Gen Z. Users highlighted how Vibez surfaced places they “wouldn’t have normally thought of” and how visuals helped them understand the mood instantly.
  • Predictability & repeatability: by tying manual ratings to automatic metrics, we can track and improve user satisfaction over time.
Research framing

What Vibez explores as a research problem.

Beyond “nice UI,” Vibez asks: how do we make LLMs and retrieval systems understand, represent, and search for vibes in a stable, safe way?

Core research questions
  • How do we encode subjective emotional language as machine-actionable labels?
  • Can a fine-tuned LLM generate vibe tags that are consistent across prompts and templates?
  • How do we align vector search, image retrieval, and safety filters so users get accurate, diverse, and safe results?
  • What is the trade-off between precision (exact vibe) and serendipity (adjacent vibes)?
Technical challenges
  • Limited emotional depth: existing datasets lack dense, high-quality labels for nuanced vibes, forcing us to blend real and synthetic data.
  • Unstable vibe tags: LLM outputs can vary across prompts, templates, and LoRA/CUDA settings between training and inference.
  • Language–image semantic gap: even with good text embeddings, image retrieval can drift without careful embedding alignment.
Key innovations
  • Novel vibe-based search engine: a structured Vibe Taxonomy translates emotional intent into embeddings, vector search, and image retrieval.
  • Synthetic data advantage: OpenAI-powered synthetic “vibe pairs” (descriptions + tags) dramatically expand emotional coverage and robustness.
  • Fine-tuned models on custom taxonomy: QLoRA and custom prompting pipelines produce stable, emotionally aligned vibe tags specialized to travel discovery.

Future roadmap

Where Vibez goes next.

The current prototype validates that “vibe-first” discovery works. The next steps focus on scaling, integrations, and turning this into a product that can live in real travel ecosystems.

Product & platform
Next steps
Deeper personalization
Incorporate user profiles, past trips, and saved vibes to personalize retrieval while maintaining safety and diversity.
Proximity & constraints
Add filters for budget, distance, and trip length — responding to early user requests like “filter by places near me.”
Real-time inventory
Connect to booking engines and inventory APIs so vibe search flows directly into bookable experiences.
B2B integrations
Discovery layer
  • Booking platforms: embed Vibez as a “mood-first search” widget to capture users at the very top of the funnel.
  • Content curators: provide tools for travel blogs and magazines to automatically tag and organize their image libraries with vibe metadata.
  • APIs & SDKs: expose the vibe tagging and retrieval pipeline so other teams can build creative experiences on top of the engine.
Vision

Vibez aims to become the default way people explore trips when they only know how they want to feel, not where they want to go — a new, emotionally intelligent discovery layer for the travel stack.
